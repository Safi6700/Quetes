## 🎯 L'ESSENTIEL (5 points)

1. **DSI** = Direction des Systèmes d'Information (transversale, spécialisée IT)
2. **Dev** = créer les logiciels | **Ops** = déployer et maintenir l'infrastructure
3. **DevOps** = réconcilier Dev et Ops → CI/CD, IaC, automatisation
4. **TSSR** = Niveau 1 de l'administration systèmes et réseaux
5. **Sécurité** = l'affaire de TOUS, pilotée par le **RSSI**

## ⚙️ APPROCHE CLASSIQUE vs DEVOPS

| Aspect | Approche Classique | Approche DevOps |
|--------|-------------------|-----------------|
| Organisation | Dev et Ops séparés | Équipes par projet |
| Déploiement | Lent, peu fréquent | Fréquent et rapide |
| Automatisation | Faible | Forte (CI/CD) |
| Infrastructure | Manuelle | IaC (Infrastructure as Code) |

## 📖 DÉFINITIONS CLÉS

| Terme | Définition |
|-------|------------|
| **IT** | Information Technology (Technologies de l'Information) |
| **SI** | Système d'Information (ensemble des ressources pour gérer l'info) |
| **CI/CD** | Continuous Integration / Continuous Deployment (automatisation) |
| **IaC** | Infrastructure as Code (infra définie par du code) |
| **Admin Systèmes** | Gestion des OS, serveurs, logiciels |
| **Admin Réseaux** | Gestion des équipements réseau |
| **Virtualisation** | Abstraction des ressources matérielles |
| **Supervision** | Surveillance de l'état du SI |
| **RSSI** | Responsable Sécurité des SI (CISO en anglais) |

## ⚠️ PIÈGE CLASSIQUE EXAMEN

```
❌ ERREUR : Confondre Dev et Ops
   → Dev = développe les logiciels
   → Ops = déploie et maintient l'infrastructure

❌ ERREUR : Penser que DevOps = un métier
   → DevOps = une APPROCHE, une culture, pas un poste

❌ ERREUR : Confondre DSI (direction) et SI (système)
   → DSI = Direction des Systèmes d'Information (l'équipe)
   → SI = Système d'Information (l'ensemble des ressources)

❌ ERREUR : Oublier que la sécurité concerne TOUT LE MONDE
   → Pas uniquement le RSSI !
```
